◆ signal()

void resumef::event_t::signal ( ) const
noexcept

触发一次信号。

如果有正在等待的协程，则最先等待的协程会被唤醒。
如果没有正在等待的协程，则信号触发次数加一。之后有协程调用wait()，则会直接返回。

注意
非协程中也可以使用。

◆ wait()

awaiter resumef::event_t::wait ( ) const
noexcept

在协程中等待信号触发。

如果信号已经触发，则立即返回true。
否则，当前协程被阻塞，直到信号被触发后唤醒。 消耗一次信号触发次数。

返回值
bool[co_await] 返回是否等到了信号
注意
只能在协程中调用。

◆ wait_for()

template<class _Rep , class _Period >
timeout_awaiter resumef::event_t::wait_for ( const std::chrono::duration< _Rep, _Period > &  dt) const
noexcept

在协程中等待信号触发，直到超时。

如果信号已经触发，则立即返回true。
否则，当前协程被阻塞，直到信号被触发后，或者超时后唤醒。 如果等到了信号，则消耗一次信号触发次数。

参数
dt超时时长
返回值
bool[co_await] 等到了信号返回true，超时了返回false。
注意
只能在协程中调用。
